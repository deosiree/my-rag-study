第一个 LangGraph 状态机

用「幻灯片」对照 firstAgent.py 里的代码，一步一步学。

如何打开：在本目录执行 python -m http.server 8765，浏览器访问 http://127.0.0.1:8765/（直接双击文件有时 Mermaid 不渲染）。

学习路线（建议顺序）

  1. State：整次运行共享的一份字典长什么样
  2. 节点：函数怎么读 state、返回什么
  3. 普通边：add_edge 固定顺序
  4. 条件边：add_conditional_edges 做分支
  5. 合流：两条分支都回到同一个节点

① State（状态）

用 TypedDict 描述「图里流转的数据结构」。


class State(TypedDict):
    topic: str
    body: NotRequired[str]
    result: NotRequired[str]

invoke({"topic": "..."}) 时只需给出必填字段；后面节点会往同一 dict 里合并 body、result。

② 节点函数

每个节点：输入完整 state，返回要更新的键值（局部 dict）。


def node_pick_topic(state: State) -> dict:
    t = state["topic"].strip()
    return {"topic": t}

不要改 state 本体，只 return {...}；LangGraph 负责合并。

③ 普通边（顺序固定）


g.add_edge(START, "pick_topic")
g.add_edge("as_question", "summary")
g.add_edge("as_fact", "summary")
g.add_edge("summary", END)

START / END 是图的入口和出口。

④ 条件边（状态机分支）

路由函数返回值必须是「分支名字」，且与第三个 dict 的 key 一致。


def route_by_question(state: State) -> Literal["as_question", "as_fact"]:
    return "as_question" if "?" in state["topic"] else "as_fact"

g.add_conditional_edges(
    "pick_topic",
    route_by_question,
    {"as_question": "as_question", "as_fact": "as_fact"},
)

⑤ 合流

as_question 与 as_fact 都连到 summary：两条路径最后汇合，再结束。

虚线边表示「条件跳转」；实线表示确定跳转（与 LangGraph 导出一致）。

动手 1：改分支规则

打开 firstAgent.py，改 route_by_question：例如用关键词「吗」、或 len(state["topic"]) 决定走哪条边。

保存后运行 python firstAgent.py 看两次 invoke 的输出是否变化。

动手 2：加一个节点

在 summary 前插入 log_step：只往 state 里写一个时间戳或计数。

记得 add_node，并把两条分支的边改成先指向新节点，再由新节点连到 summary。

常用命令

  • python firstAgent.py --mermaid — 终端打印图
  • python firstAgent.py --write-mermaid workflow_slides/graph.mmd — 更新流程图文件
